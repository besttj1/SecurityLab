# 🛡️ SecurityLab: CompTIA Security+ (SY0-701) Implementation
**Author:** Julius Tijani  
**Objective:** Bridging the gap between Security+ theory and practical automation.

## 🚀 Overview
This repository serves as a technical portfolio for my CompTIA Security+ certification journey. Unlike traditional study notes, this lab focuses on **Security Automation**—using Python and Node.js to implement the concepts found in the DoD 8140 / DCWF framework.

## 🛠️ Technical Stack
- **Languages:** Python 3.x, Node.js (Express)
- **Frameworks:** CompTIA Security+ (SY0-701)
- **Lab Environment:** TryHackMe, Kali Linux, VS Code

## 📁 Repository Structure
Each folder corresponds to a core Security+ Domain:

### [Domain 1.0: Threats, Attacks, and Vulnerabilities](./Domain1.0-General-Security-Concepts)
- **Malware Signature Matcher (Python):** A script that simulates antivirus detection by matching file hashes against a known malicious database.
- **Log Scraper:** Extracts suspicious IP addresses from server logs to identify brute-force attempts.

### [Domain 3: Architecture and Design](./Domain3)
- **Security Header Checker (Node.js):** An automated tool to verify if web applications are using HSTS, X-Frame-Options, and CSP headers.
- **Hashing Utility:** Implements SHA-256 and MD5 hashing for data integrity verification.
